SCSS @use @forward

개요

@use와 @forward는 기존의 @import를 대체하는 새로운 모듈 시스템입니다.
이 두 규칙을 함께 사용하여 스타일을 모듈화하고 관리하는 방식은 프로젝트를 더 체계적이고 효율적으로 만듭니다.

주요 내용

  • 네임스페이스(Namespacing) 사용
  • 범위(Scope) 제한
  • 모듈 내보내기

공통 파일 정의


// _colors.scss
$primary: blue;

// _sizes.scss
$primary: 16px;

@import 예제 1


// style.scss
@import 'colors';
@import 'sizes';

.element {
  // $primary는 최종적으로 '16px'의 값을 가집니다.

  color: $primary;
  font-size: $primary;
}

@import 예제 2


// style.scss
.element {
  color: $primary;
  font-size: $primary
}

// main.scss
@import 'colors'
@import 'sizes'
@import 'style'

속성 설명

  • @import는 모든 변수를 전역으로 가져오기 때문에 이름이 같은 변수가 충돌합니다.
  • 나중에 가져온 파일의 값으로 덮어쓰기 때문에 개발자가 의도한 색상이 아닌 크기가 적용되는 전역 오염 문제가 발생합니다.

@use 예제


// style.scss
@use 'colors';
@use 'sizes';

.element {
  color: colors.$primary;
  font-size: sizes.$primary;
}

// main.scss
@use 'style';

속성 설명

  • @use는 가져온 파일 이름을 네임스페이스로 자동 부여하여 변수 이름을 격리합니다. 변수 이름이 같더라도 네임스페이스로 구분되기 때문에 충돌이 발생하지 않습니다.
  • 변수를 사용하려면 반드시 해당 파일에서 @use규칙을 선언해야 합니다.
    필요한 곳에서 필요한 것만 명확한 이름으로 가져와 사용하도록 강제하여 코드를 훨씬 안전하고 관리하기 쉽게 만듭니다.

@forward 예제: 개별 파일 정의


// lib/_colors.scss
$primary: blue;

// lib/_mixin.scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

@forward 예제: 묶음 파일 생성


// lib/_index.scss
@forward 'colors';
@forward 'mixin';

속성 설명

  • 프로젝트에 간단한 색상 변수와 믹스인이 있다고 가정합니다.
  • lib폴더 안에 _index.scss파일을 생성하여 위 두 파일을 @forward로 묶어줍니다.

@forward 예제: 묶음만 @use하여 사용


// style.scss
@use 'lib';

.element {
  @include lib.flex-center;

  color: lib.$primary;
}

속성 설명

  • _color.scss나 _mixin.scss를 일일히 @use할 필요 없이 묶음 파일인 lib폴더만 @use하면 됩니다.
  • @forward는 style.scss와 개별 파일 사이의 통로 역할을 합니다.
  • 개발자는 lib/_index.scss만 관리하면 되고 style.scss는 깔끔하게 lib모듈만 가져와서 사용하게 됩니다. 이 방식이 모듈화의 핵심입니다.

나의 생각 및 응용

SCSS를 처음 사용할 때는 @import로 모든 스타일을 전역에서 공유하는 방식으로만 사용했습니다.
하지만 @use와 @forward를 학습한 후 스타일을 모듈화 하고 네임스페이스로 관리하면서 코드를 더 안전하고 체계적으로 확장할 수 있게 되었습니다.

이를 통해 유지보수성과 확장성을 고려한 구조적 스타일 설계의 중요성을 느꼈습니다.

기술 적용 사례

이 기술은 SCSS web 페이지 프로젝트에 활용했습니다.
