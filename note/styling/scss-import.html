SCSS @import

개요

SCSS에서 @import는 여러 SCSS 파일을 하나로 합쳐주는 기능입니다.
이를 통해 스타일을 기능별, 파트별로 나누어 관리하고, 최종 빌드 시 한 CSS 파일로 컴파일할 수 있게 도와줍니다.

주요 내용

  • 여러 파일 가져오기
  • 파일 분할(Partials)

예제 코드


/*main.scss*/

@import "variables";
@import "reset";
@import "mixin";
@import "button";
@import "components";
@import "header";
@import "layout";
@import "footer";

속성 설명

  • 위처럼 작성하면 각각 파일의 내용을 불러와서 현재 파일에 포함시킵니다.
  • 보통 main.scss에서 모든 SCSS 파일을 @import하여 중앙에서 관리합니다.

컴파일 전


...

  ├─ scss
  │  ├─ variables.scss
  │  ├─ reset.scss
  │  ├─ mixin.scss
  │  ├─ button.scss
  │  ├─ components.scss
  │  ├─ header.scss
  │  ├─ layout.scss
  │  ├─ footer.scss
  │  └─ main.scss

...

컴파일 후


...
  ├─ css
  │  ├─ variables.css
  │  ├─ reset.css
  │  ├─ mixin.css
  │  ├─ button.css
  │  ├─ components.css
  │  ├─ header.css
  │  ├─ layout.css
  │  ├─ footer.css
  │  └─ main.css
  ├─ scss
  │  ├─ ...
...

속성 설명

  • 프로젝트 규모가 커지면 파일들을 각 기능과 부분으로 나눠 유지 보수가 쉽도록 관리하게 됩니다.
  • 파일이 점점 많아지는데, 모든 파일이 컴파일 시 각각의 .css 파일로 나눠서 저장된다면 불필요한 결과물이 생기게 됩니다.

컴파일 전


...
  ├─ scss
  │  ├─ _variables.scss
  │  ├─ _reset.scss
  │  ├─ _mixin.scss
  │  ├─ _button.scss
  │  ├─ _components.scss
  │  ├─ _header.scss
  │  ├─ _layout.scss
  │  ├─ _footer.scss
  │  └─ main.scss
...

컴파일 후


...

  ├─ css
  │  └─ main.css
  ├─ scss
  │  ├─_variables.scss
  │  ├─_reset.scss
  │  ├─ ...
  │  ├─ _footer.scss
  │  └─ main.scss

...

속성 설명

  • 파일 이름이 언더스코어(_)로 시작하면 SCSS가 partial 파일(조각 파일)로 인식합니다.
  • 단독으로 컴파일되지 않고, 오직 @import로만 포함되도록 처리됩니다.

나의 생각 및 응용

CSS에서는 각 기능별로 스타일을 나눠서 작성해도, HTML에 <link>로 각각 연결하거나 직접 병합해야 하는 번거로움이 있었습니다.

반면 SCSS에서는 @import를 사용해 필요한 스타일 파일들을 하나의 메인 파일로 불러올 수 있어서 관리가 훨씬 편리했고,
컴파일 시 단일 CSS 파일로 생성되어 배포나 유지보수에도 효율적이라는 장점을 느꼈습니다.

기술 적용 사례

이 기술은 스타벅스 페이지 프로젝트의 SCSS파일에 활용했습니다.
