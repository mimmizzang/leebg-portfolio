SCSS Map과 List를 활용한 동적 레이아웃 생성

개요

만약 수십 개의 아이템 크기나 배경 이미지를 CSS로 일일이 지정해야 한다면 이는 CSS 코드를 비대하게 만들고 유지보수를 어렵게 합니다.
SCSS의 Map과 List, @each 반복문을 사용하여 데이터를 구조화 하고 모든 스타일을 자동으로 생성하여 중복되는 스타일을 최소화해 효율적으로 관리합니다.

주요 내용

  • Map 구조 정의
  • @each와 map.get
  • List와 중첩 루프

예제 코드


@use "sass:map";
@use "sass:list";

$articles: (
  "1": (
    width: 700px,
    height: 400px,
    bg: "700x400.png",
    inner: (
      "MerryChristmas",
    ),
  ),
  "5-1": (
    width: 490px,
    height: 240px,
    bg: "490x240-1.png",
    inner: (
      "Programmer",
    ),
  ),
  "7": (
    width: 1000px,
    height: 371px,
    bg: "1000x371.png",
    inner: (
      "Yours",
    ),
    inner-size: (
      686px,
      60px,
    ),
    pos-center: true,
    text-shadow: true,
  ),
// ... (생략) ...
);

속성 설명

  • $articles는 데이터베이스 역할을 하며 모든 아이템의 속성을 구조화 한 키-값 쌍으로 저장합니다.
  • 각 아이템 내부에 또 다른 Map을 사용하여 속성을 관리하는 데이터 중첩 방식을 보여줍니다.
  • inner-size처럼 두 개 이상의 값을 하나의 키에 묶어 저장하고 나중에 순서대로 추출할 데이터는 Map이 아니라 list로 저장했습니다.

예제 코드


@each $key, $props in $articles {
  .article--#{$key} {
    width: map.get($props, width);
    height: map.get($props, height);
    float: left;
    position: relative;
    overflow: hidden;

    @if map.has-key($props, bg) {
      $bg-img: map.get($props, bg);
      background-image: url(../img/#{$bg-img});
    }
// ... (생략) ...
  }
}

속성 설명

  • @each는 Map 반복의 기본 구문입니다. $key에는 항목 이름이 $props에는 해당 항목의 속성을 담은 중첩 Map이 할당됩니다.
  • #{$key} 변수를 사용하여 CSS 선택자를 동적으로 생성합니다.
  • map.get을 이용하여 Map에서 값을 꺼냅니다. $props Map에서 키에 해당하는 값을 가져와 CSS 속성 값으로 사용합니다.

예제 코드


// ... @each 루프 계속 ...

    @if map.has-key($props, inner) {
      @each $inner-name in map.get($props, inner) {
        .#{$inner-name} {
          position: absolute;

          @if map.has-key($props, inner-size) {
            $size: map.get($props, inner-size);
            width: list.nth($size, 1);
            height: list.nth($size, 2);
          }
// ... (생략) ...
        }
      }
    }

// ... @each 루프 종료 ...

속성 설명

  • 중첩 루프를 사용하여 Map 안에 저장된 inner List의 각 요소에 접근합니다.
  • inner-size 키에 해당하는 List 데이터 묶음 전체를 $size 변수에 할당합니다.
  • list.nth를 이용하여 List에서 값을 꺼냅니다. Map이 키 이름으로 접근하는 것과 달리 List는 인덱스 번호를 사용하여 원하는 값을 추출합니다.

컴파일 후

  overflow: hidden;

.article--7 {
  width: 1000px;
  height: 371px;
  float: left;
  position: relative;
  overflow: hidden;
  background-image: url(../img/1000x371.png);

.article--7 .Yours {
  position: absolute;
  width: 686px;
  height: 60px;
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  text-shadow: 2px 2px 2px black;
}
// ... (생략) ...

나의 생각 및 응용

$articles Map을 활용해 반복되는 .article--N 스타일을 자동 생성했습니다.
Map과 List, @each를 통해 중복 코드를 줄이고 유지보수성을 높이는 구조적 스타일링을 적용할 수 있었습니다.

이를 통해 스타일 코드도 하나의 데이터처럼 관리할 수 있다는 점을 체감했습니다.

기술 적용 사례

이 기술은 SCSS web 페이지 프로젝트에 활용했습니다.
